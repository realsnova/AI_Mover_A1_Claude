# Phase 5 設計文件：出廠品管、強固化、家長說明

> 修改對象：`單字冒險.html` + 新增 `家長使用說明.md`

## 1. 目標

交付前的最終品管：內容正確性全掃描、極端情況防護、跨裝置版面、真人兒童測試腳本、給家長的一頁說明。

## 2. 自動化內容掃描（沙盒 Python 腳本）

寫腳本掃描內嵌後的 WORDS_ALL，輸出問題清單並逐一修正：

1. distractors 不含正解（含大小寫與變形，例：distractor "skate" vs 正解 "ice skates"）
2. 每字 3 個干擾字、無重複
3. 例句文法抽查：不得出現 Movers 超綱結構（現在完成式 have seen、被動語態、will 未來式）——用關鍵詞規則掃 + 人工抽查 30 句
4. sentence 內含正解字（填空題可挖空）；不含者列出改寫
5. emoji 欄位再掃一次 Unicode 13+
6. 中譯抽查：隨機 30 字人工核對台灣用語（例：鳳梨不是菠蘿）

## 3. 程式強固化

- **TTS fallback**：`speechSynthesis.getVoices()` 無英文語音時：聽音題（B）自動停用、學習卡與答題的發音鍵顯示為灰色加提示「此電腦沒有英文語音」——不可讓按了沒反應
- **語音載入時序**：Chrome 的 getVoices 需要 `voiceschanged` 事件後才有值 → speak() 前如果 voices 空，掛一次性 voiceschanged 重試
- **存檔保護**：version 檢查 + JSON.parse try/catch + 壞檔備份重置（Phase 4 已做，此處驗證）
- **離線模式**：無網路時 sprite fallback 👾（已有）、Twemoji 保留原生 emoji（已有）→ 驗證整個遊戲離線仍可玩（聽說讀寫僅圖示變醜）
- **防連點**：answering 鎖已有 → 快速連點答案鈕/翻頁鈕測試無重複計分
- **音效開關**：新增 🔇/🔊 切換鈕（頂欄），設定存檔（教室情境需要靜音）

## 4. 版面與可用性檢查

- 1366×768（學校電腦）：無水平捲軸、按鈕不重疊、單畫面可見主要操作
- iPad 直式 768×1024 與橫式：觸控目標 ≥ 64px、拼字題字母塊不換行爆版（8 字母 + 清除鈕）
- 手機 390×844（縮小支援，非主要目標）：可玩即可
- 檢查法：Chrome DevTools device emulation 各尺寸截圖逐頁檢查（標題/地圖/學習/四題型/道館/圖鑑/商店/儀表板/劇情）
