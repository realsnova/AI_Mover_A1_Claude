## Phase 4+ 新增功能（戰鬥升級 + 存檔強化）

### 戰鬥舞台升級（精緻奇幻 RPG 風格）

- **天空背景**：3 種戰場各升級成奇幻天空（高空冷色 → 地平線金色魔法光），加遠景山脈/城堡剪影、神聖光束、飄浮火星粒子、暗角營造景深。
- **召喚舞台**：敵人腳下發光魔法陣＋旋轉符文環＋脈動光暈＋接地陰影，出場落地演出。
- **敵人血條**：金框血條隨戰鬥推進扣血（綠→黃→紅），視覺反饋「真正打倒牠」。
- **打擊感（juice）**：答對時 → 斬擊光刃＋衝擊光環＋9 道火花爆散＋畫面震動＋強化命中閃光＋浮動傷害數字。
- **連擊系統**：連續答對跳出「🔥 N 連擊！」徽章＋浮動「命中 ×N」數字；答錯重置。
- **奇幻邊框**：舞台加金色雙鑲邊＋內發光，提升質感。

### 存檔機制升級（防進度丟失）

**四層自動存檔（保險絲設計）**：
1. `beforeunload` → 關頁/重新整理自動存檔
2. `visibilitychange` → 切分頁/縮小視窗自動存檔（教室/家裡常見）
3. 每 15 秒定時存檔（安全網）
4. 每次 `switchScreen()` 都存（畫面切換時存）

**手動存檔點（可回頭還原）**：
- 標題畫面新增「💾 存檔」按鈕，開啟彈窗顯示目前進度摘要＋存檔時間
- 建立獨立存檔槽（`SLOT_KEY`），與自動存檔（`SAVE_KEY`）分開
- 支援「存檔碼」機制：Base64 編碼進度 JSON，可複製備份 → 防 localStorage 被清空／轉移到其他裝置

**載入存檔**：
- 標題畫面新增「📂 載入存檔」按鈕，顯示存檔槽的時間與內容摘要
- 一鍵還原（有確認對話防誤觸）
- 支援用「存檔碼」匯入（包含格式驗證＋容錯）

**重看故事**：
- 標題畫面新增「📖 重看故事」按鈕，不受 `storySeen` 限制，強制重播開場劇情（含序曲配樂）
- 播完或跳過都回到地圖

### 相關常數與方法

```javascript
const SLOT_KEY = "wordAdventure_v1_slot";  // 手動存檔點（獨立於自動存檔）

// 核心方法
Game.save()               // 更新自動存檔；已加空狀態防呆
Game.replayStory()        // 強制重播開場
Game.manualSave()         // 建立手動存檔點 + 時間戳
Game.encodeSave(st)       // Base64 編碼進度為存檔碼
Game.decodeSave(code)     // 反編碼存檔碼（含格式驗證 + 容錯）
Game.openSaveDialog()     // 開啟存檔彈窗
Game.openLoadDialog()     // 開啟載入彈窗
Game.applyState(obj)      // 還原進度對象並跳地圖
```

## 4. 關鍵技術決策（不要走回頭路）

1. **Twemoji CDN**：cdnjs 的 twemoji 是 404！正確引用：
   - script：`https://cdn.jsdelivr.net/npm/@twemoji/api@15.1.0/dist/twemoji.min.js`
   - 圖檔 base：`https://cdn.jsdelivr.net/gh/jdecked/twemoji@15.1.0/assets/`（folder: svg, ext: .svg）
2. **emoji 政策**：資料中只放 Windows 10 可顯示的 emoji（Unicode ≤12）；無合適 emoji 的字留空字串 → 遊戲以白底中文字卡顯示（zh-fallback）。Twemoji 是加值層，離線時退回原生 emoji 也不能破版
3. **寶可夢 sprite**：`https://raw.githubusercontent.com/PokeAPI/sprites/master/sprites/pokemon/{id}.png`（shiny 加 /shiny/）；img onerror 全域備援顯示 👾
4. **音效**：全部 Web Audio 程式合成（tone/sfxCorrect/sfxWrong/sfxCoin/jingleVictory），無外部音檔、無版權疑慮
5. **發音**：SpeechSynthesis en-US rate 0.85；Chrome 需使用者點擊過頁面後才允許（實際使用必然滿足）
6. **單一檔案原則**：所有 CSS/JS/資料內嵌 `單字冒險.html`，使用者雙擊即玩（file:// 環境，fetch 不可用）
7. **版權**：寶可夢素材僅限個人/課堂非商業使用，不得公開部署（規劃文件第 0 章）
8. **存檔策略（Phase 4+）**：
   - 自動存檔用 `SAVE_KEY`，四層保險（beforeunload/visibilitychange/定時/換畫面）
   - 手動存檔用 `SLOT_KEY`（獨立槽），可設時間戳、可回頭
   - 存檔碼用 Base64（支援備份/轉移），`decodeSave()` 含容錯（壞碼返回 null）
   - `save()` 加空狀態防呆，標題畫面不會誤寫

## 5. 開發環境已知問題與解法（重要，會踩坑）

1. **outputs 掛載同步會卡住舊版**：`單字冒險.html` 被多次 Edit 後，Linux 沙盒掛載可能一直讀到舊內容。
   - 驗證實際檔案一律用 Read/Grep 工具（Windows 側，永遠是最新）
   - 需要在沙盒處理最新內容時：用 Write 另存新檔名（新檔案會同步），用完刪除
2. **瀏覽器擴充不能開 file:// 網址**。實測方法（已驗證可行）：
   - navigate 到 `https://example.com` → javascript_tool 建立 `<input type="file">` + FileReader 存 window.__HTML → `file_upload` 工具上傳 單字冒險.html → `document.open(); document.write(window.__HTML); document.close()`
   - 注入後 localStorage/語音/CDN 都正常，可完整測試
3. **截圖座標 ≠ 視窗座標**：截圖可能縮放（如 1568 寬 vs 實際 1920）。點擊一律用 read_page 的 ref，不要用截圖座標
4. **javascript_tool 45 秒超時**：長流程（自動答題迴圈）要拆成多次呼叫，每次 ≤ 3 題；超時不代表頁面壞了，下一次呼叫讀狀態即可
5. **speechSynthesis 測試**：先用 computer 工具真實點擊頁面一次，再測 utterance 的 onstart/onend 事件；直接腳本呼叫會得到 not-allowed（是政策不是 bug）
6. **document.write 注入後 DOMContentLoaded 可能不觸發**：主程式的 initTwemoji 已改為 readyState 判斷，任何載入方式都會初始化（勿改回）
7. **存檔碼測試（Phase 4+）**：已驗證 Base64 編解碼往返、中文字正確、壞碼安全回傳 null、純 JSON 容錯。瀏覽器實機測試清單見下方。

## 6. 標準開發循環（每個 Phase 照做）

1. 讀對應的 `PhaseN_設計與驗證清單.md`
2. 用 Edit 修改 `單字冒險.html`
3. 新程式區塊抽出到 /tmp 做 `node --check`（完整檔案檢查用「另存新檔名→沙盒 node --check」）
4. 依第 5 節方法把檔案注入 Chrome，逐項執行該 Phase 驗證清單（真實點擊+截圖+JS 斷言）
5. 發現問題 → 修正 → 重新注入重測
6. 全過後：更新驗證清單 .md 紀錄結果 → present_files 交付 → 告知使用者測試重點
