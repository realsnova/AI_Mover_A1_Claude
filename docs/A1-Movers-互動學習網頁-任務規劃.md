# A1 Movers 單字冒險：互動學習網頁任務規劃

> 目標對象：台灣國小三年級學生（8–9 歲）
> 內容基準：Cambridge English A1 Movers（2018 年版官方單字表，約 550 字）
> 遊戲主題：寶可夢式「收服／進化／對戰」+ 勇者鬥惡龍式 RPG 冒險

## 1. 產品概述

### 1.1 一句話定義
一款單機網頁遊戲：學生扮演「單字訓練家」，在地圖上冒險，透過答對 A1 Movers 單字題來收服、養成、進化寶可夢，並挑戰道館 Boss。

### 1.2 核心學習循環（每 3–5 分鐘一輪）
```
遇見野生寶可夢 → 單字題挑戰（聽/看/選/拼） → 答對收服/獲得經驗
→ 寶可夢升級進化 → 解鎖新地圖區域（新單字主題） → 挑戰道館（單元總測驗）
```

### 1.3 成功指標
- 單次使用 15–20 分鐘內完成一個主題關卡
- 每個單字至少以 3 種題型出現（間隔重複）
- 學生不需家長協助即可獨立操作（見第 4 章 UI/UX 準則）

## 2. 教材內容架構

### 2.1 內容來源

官方資源（單字與題型的唯一基準）：

- A1 Movers 官方單字表（2018 起適用）：https://www.cambridgeenglish.org/Images/506166-starters-movers-flyers-word-list-2018.pdf
- A1 Movers Word List Picture Book（主題式圖解字彙書，10 大主題情境）：https://www.cambridgeenglish.org/images/396159-yle-movers-word-list-picture-book-2018.pdf
- 官方考題結構：Listening 5 parts / Reading & Writing 6 parts / Speaking 4 parts —— 題型設計參照此結構簡化

非官方／官方授權教材（用於例句、故事情境、練習題型參考，不直接複製內文）：

- Kid's Box (Cambridge)：Level 3–4 對應 Movers，主題單元順序可參考
- Fun for Movers (4th ed.)：官方指定備考書，題型變化參考
- Storyfun for Movers：故事化學習，作為「劇情關卡」文本風格參考
- Fun Skills 3–4：聽說讀寫分項練習參考

### 2.2 單字資料結構（AI 開發時的核心資料格式）

全部約 550 字建成 `wordlist.json`，schema 如下：

```json
{
  "id": "m-013",
  "word": "dolphin",
  "zh": "海豚",
  "topic": "animals",
  "pos": "noun",
  "level": "movers",
  "sentence": "A dolphin can jump out of the water.",
  "sentence_zh": "海豚可以跳出水面。",
  "image": "img/dolphin.png",
  "distractors": ["shark", "whale", "kangaroo"]
}
```

### 2.3 主題分區（= 遊戲地圖區域），涵蓋官方單字表全部主題

| # | 地圖區域 | 官方主題 | 約略字數 |
|---|---------|---------|--------|
| 1 | 新手草原 | Animals 動物 | 45 |
| 2 | 美食村 | Food & drink 食物飲料 | 50 |
| 3 | 身體迷宮 | The body & face / Health | 35 |
| 4 | 學園城 | School 學校 | 40 |
| 5 | 運動競技場 | Sports & leisure | 55 |
| 6 | 家園小鎮 | The home / Family & friends | 60 |
| 7 | 天氣高塔 | Weather / The world around us | 45 |
| 8 | 城市大道 | Places & directions / Transport | 55 |
| 9 | 時光神殿 | Time / Numbers 1–100 | 40 |
| 10 | 冠軍之路 | Work / Clothes / 形容詞動詞綜合 | 125 |

每區含：野外遭遇戰（新字學習）→ 3 個小關（練習）→ 道館戰（單元測驗，答對 8/10 過關）。

### 2.4 題型設計（對應官方考試能力，全部做成遊戲互動）

1. **看圖選字**（Reading Part 1 簡化）：出現圖片，從 3 個選項點選正確單字 → 攻擊命中
2. **聽音選圖**（Listening Part 4 簡化）：播放單字/短句發音（Web Speech API），點選正確圖片
3. **拼字組合**：把打散的字母方塊拖曳（或依序點擊）拼出單字 → 投出精靈球收服
4. **句子填空**：例句挖空，三選一 → 對戰技能升級
5. **快問快答**（道館 Boss 戰）：混合以上題型限時作答

## 3. 遊戲化系統設計

### 3.1 寶可夢機制

- **收服**：每個主題區配置 6–8 隻寶可夢（用 PokeAPI sprite）。答對「拼字組合」題即收服，存入圖鑑。
- **進化**：同一單字在不同天答對 3 次（間隔重複）→ 該區寶可夢進化一階。進化 = 複習的視覺回饋。
- **對戰**：野怪戰答錯不懲罰扣血，只顯示「Miss!」並立刻給正解重試一次（低挫折設計）；道館戰才計分。
- **圖鑑**：收集頁 = 學習進度頁。每隻寶可夢卡片背面就是它綁定的單字卡（字、圖、音、例句）。

### 3.2 勇者鬥惡龍機制

- **像素風大地圖**：10 個區域用步行點選移動，DQ 式開場「勇者啊，去收集單字的力量吧！」
- **升級音效與儀式感**：過關播放 DQ 風格升級旋律（可用 8-bit 合成音，避免直接使用原版音源）
- **金幣商店**：答題賺金幣，購買裝飾道具（帽子、背景）——只買外觀，不買能力，避免 pay-to-skip 學習

### 3.3 激勵與節奏

- 每日登入獎勵（連續登入給稀有寶可夢遭遇）
- 學習節奏：一次野外遭遇 = 5 個新字 + 5 個舊字混合，符合工作記憶容量
- 錯題自動進入「傷寶可夢中心」佇列，優先在下次遭遇出現

## 4. UI/UX 準則（針對國小三年級電腦操作能力）

三年級學生的操作特性：滑鼠點擊熟練、拖曳不穩定、打字很慢、注音輸入尚可但英打幾乎不會、閱讀中文為主。

**必須遵守：**

1. **一律點選優先**：所有題型都可用「純點擊」完成；拼字題用點擊字母方塊，不強制拖曳、不要求鍵盤輸入
2. **大目標區**：所有可點擊元素 ≥ 64×64px，間距 ≥ 16px
3. **介面中文、學習內容英文**：按鈕、說明、回饋用繁體中文（可加注音給較弱讀者做選項）；單字、例句、語音為英文
4. **零文字依賴導航**：主要導航用圖示 + 顏色（圖鑑 = 紅色球、地圖 = 綠色、商店 = 金幣），配少量中文
5. **即時語音回饋**：每題答完自動朗讀正確單字（SpeechSynthesis, en-US, rate 0.85）
6. **不出現失敗畫面**：答錯只有「再試一次！」，永遠可重試；避免倒數計時壓力（僅 Boss 戰有寬鬆計時）
7. **單一畫面單一任務**：一個畫面只有一個主要行動，無多層選單
8. **自動存檔**：進度自動保存，無帳號、無登入流程（單機版存 localStorage；若做成 Claude Artifact 則改用 React state + 匯出進度碼）
9. **響應式**：支援學校電腦 1366×768 與平板觸控

## 5. 技術架構建議

- **形式**：純前端單頁應用，單一 HTML 檔（HTML + CSS + Vanilla JS 或 React），無後端、無資料庫
- **語音**：Web Speech API（SpeechSynthesis）發音，零成本免 API key
- **圖片**：寶可夢 sprite 取自 PokeAPI 靜態 CDN；單字圖用 emoji 或開源圖示（先用 emoji 快速開發，後換插圖）
- **音效**：8-bit 風格用 Web Audio API 程式合成（避免版權音源）
- **存檔**：localStorage（本機部署時）
- **資料分離**：`wordlist.json`、`monsters.json`、`maps.json` 與遊戲引擎分離，內容可獨立擴充

## 6. 分階段開發任務（每階段附可直接使用的 AI 開發 Prompt）

### Phase 0：內容資料建置（1 個 AI 會話）

任務：產出完整 550 字 `wordlist.json`。

> **AI Prompt 0**
> 「你是劍橋兒童英檢教材編輯。請依據 Cambridge A1 Movers 2018 官方單字表，產出完整單字 JSON 陣列。Schema：{id, word, zh(繁體中文), topic, pos, sentence(A1程度例句,≤8字), sentence_zh, distractors(3個同主題干擾字)}。主題分類使用：animals, food, body, school, sports, home_family, weather_world, places_transport, time_numbers, mixed。請分 10 個主題逐批輸出，每批完成後我會說『繼續』。例句須符合 Movers 文法範圍（現在式、現在進行式、過去簡單式、比較級、have to、want to）。」

驗收：字數 ≥ 520、每字有中譯與例句、干擾字不重複於正解。

### Phase 1：遊戲核心引擎（MVP，先做 1 個主題區）

任務：單一 HTML 檔，含大地圖（僅開放「新手草原」）、野外遭遇戰、看圖選字 + 聽音選圖兩種題型、收服動畫、圖鑑頁、localStorage 存檔。

> **AI Prompt 1**
> 「請建立單一 HTML 檔的兒童英語學習遊戲，對象為台灣國小三年級學生。需求：(1) 像素 RPG 風格大地圖，點選區域進入關卡；(2) 遭遇戰畫面：上方顯示寶可夢 sprite（用 https://raw.githubusercontent.com/PokeAPI/sprites/master/sprites/pokemon/{id}.png），下方為題目區；(3) 題型 A：顯示英文單字圖示（emoji），三個大按鈕選單字；題型 B：SpeechSynthesis 播放單字，點選正確 emoji；(4) 答對播放攻擊動畫並累積收服進度，5 題全對觸發精靈球收服動畫並寫入圖鑑；(5) 答錯顯示『Miss! 再試一次』並朗讀正確答案，可無限重試；(6) 介面全繁體中文、按鈕 ≥64px、僅用點擊操作；(7) 進度存 localStorage。附上這批單字資料：[貼入 Phase 0 的 animals 主題 JSON]。」

驗收：8 歲兒童可在無指導下完成一場遭遇戰並收服一隻寶可夢。

### Phase 2：題型擴充與道館系統

任務：加入拼字組合（點擊字母方塊）、句子填空題型；道館 Boss 戰（10 題混合、8 題過關、獲得徽章）；金幣與商店（外觀道具）。

> **AI Prompt 2**
> 「在現有遊戲加入：(1) 拼字題：單字字母打散成方塊，點擊依序組字，可點『清除』重來，提供第一個字母提示；(2) 填空題：顯示例句挖空與中文翻譯，三選一；(3) 道館戰：進入條件為該區收服 6 隻，10 題混合題型、寬鬆計時（每題 20 秒）、答對 8 題獲得徽章與 DQ 風格勝利音樂（Web Audio API 合成 8-bit）；(4) 金幣系統與商店：每答對 +2 金幣，商店販售訓練家帽子與背景，純裝飾。維持既有 UI 準則。」

### Phase 3：間隔重複與進化系統

任務：實作 SRS（簡化版 Leitner 3 盒制）、錯題優先復現、寶可夢進化綁定複習次數、每日登入獎勵。

> **AI Prompt 3**
> 「加入學習科學機制：(1) 每個單字記錄 box(1–3) 與 lastSeen；答對升盒、答錯降回 box1；每場遭遇戰自動組卷 = 5 個 box1 新字 + 3 個到期複習字 + 2 個錯題字；(2) 單字在不同日期答對 3 次後，綁定的寶可夢進化（換 sprite），播放進化動畫；(3) 每日首次登入送稀有遭遇。所有邏輯寫成獨立 srs.js 模組並附註解。」

### Phase 4：內容全量導入與 10 區地圖

任務：導入全部 550 字與 10 個地圖區域，區域依序解鎖（取得前一區徽章）；加入 DQ 式開場劇情與結局（集滿 10 枚徽章成為「單字冠軍」）。

> **AI Prompt 4**
> 「將完整 wordlist.json（550 字、10 主題）與 maps.json 導入，實作：區域鎖定/解鎖、總進度儀表板（給家長/老師看：各主題熟練度百分比）、開場像素劇情對話框（DQ 風格逐字顯示）、集滿徽章的結局動畫。確保 550 字資料下 localStorage 存檔仍 < 1MB。」

### Phase 5：測試與優化

任務：真人兒童測試 + 自動化檢查。

> **AI Prompt 5**
> 「請對此遊戲做出廠檢查並修復：(1) 檢查所有 550 字的 distractors 不含正解、例句無超綱文法；(2) 所有按鈕在 1366×768 與 iPad 直式下 ≥64px 且不重疊；(3) SpeechSynthesis 在無 en-US 語音包時的 fallback；(4) localStorage 損毀時的重置保護；(5) 產出一頁『家長使用說明』。」

真人測試腳本：找 2–3 名三年級學生，觀察是否能（a）不問人完成第一場戰鬥（b）知道去哪裡複習（c）20 分鐘後仍想繼續玩。任一失敗即回修 UI。

## 7. 導入 AI 工具的操作方式

1. **Claude Code / Cursor**：把本文件放專案根目錄命名 `CLAUDE.md`（或 `.cursorrules` 引用），再逐階段貼入 Prompt 0–5。每階段完成先驗收再進下一階段。
2. **Claude.ai / ChatGPT 網頁版**：Phase 0 內容生成直接在對話完成；Phase 1–4 每階段開新對話，附上「目前完整程式碼 + 該階段 Prompt」。
3. **原則**：一次只做一個 Phase；每個 Phase 結束用第 6 章驗收條件測試；資料（json）與程式分開請 AI 生成，避免單次輸出過長截斷。
