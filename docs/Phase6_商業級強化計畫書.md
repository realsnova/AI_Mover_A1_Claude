## 二、「商業級」驗收標準（本計畫的成功定義）

1. **離線可玩**：斷網（第二次開啟）後，全流程可正常遊玩，怪獸圖與 emoji 不破圖
2. **可安裝**：手機可「加入主畫面」成為全螢幕 App（PWA installable）
3. **零單點故障**：CDN 掛掉、GitHub 被擋、TTS 無英文語音——任一情況遊戲仍可玩
4. **鍵盤可完整操作**一場戰鬥；Lighthouse Accessibility ≥ 90
5. **細節無瑕**：無 404、無 console 錯誤、有 favicon、有音量控制

## 四、分階段執行計畫

### Phase 0：基線保存（工作量：極小）
- 提交並推送目前兩批變更（音樂外部化、3 連接詞），作為安全回退點。
- **驗收**：git 乾淨、GitHub Pages 線上版更新。

### Phase A：專業細節打磨（工作量：小）
| 工作項 | 說明 |
|--------|------|
| A1 favicon | emoji SVG data URI 內嵌（🎮 或 ⚡），消除 404；為 Phase B 順便產出 192/512 PNG icon |
| A2 補 emoji | fly🪰、rock🪨、roller skates/skating🛼、lift/elevator🛗、toothbrush🪥 等 8 字；配圖原則：**不重複、不誤導，配不出就保留中文備援**（需檢查 WORD_EMOJI 衝突） |
| A3 音訊設定 | 頂欄 ⚙️ 按鈕 → 面板：背景音樂／音效／單字發音三個開關，localStorage 記憶（獨立於存檔） |

- **驗收**：載入零 404；設定可靜音且重啟記憶；三處資料檔（source/wordlist/HTML）同步。

### Phase B：PWA 化 — 離線可玩＋可安裝（工作量：大）★ 本計畫核心
| 工作項 | 說明 |
|--------|------|
| B1 manifest | name/icons/theme/display:standalone/start_url，加入主畫面即全螢幕 App |
| B2 Service Worker | 預快取自家資產（HTML/MP3/icon）；**runtime cache-first** 快取 PokeAPI 怪獸圖與 Twemoji CDN（首次上線瀏覽後即離線可用；不把任天堂素材 commit 進 repo，迴避再散布問題）；版本化快取 + 「有新版本」更新提示 |
| B3 破圖後備 | 怪獸圖 onerror → emoji 佔位符；確保首次離線也不開天窗 |
| B4 TTS 偵測 | 無英文語音時自動停用「聽音選圖」題型（改出其他 3 種），聽力按鈕顯示提示 |

- **驗收**：斷網重開全流程可玩；Lighthouse PWA installable；手機加入主畫面實測。
- **風險**：中文檔名在 SW cache 需 encodeURI；Twemoji opaque response 快取需設條目上限；GitHub Pages 為 HTTPS ✓。

### Phase C：無障礙與鍵盤操作（工作量：中）
| 工作項 | 說明 |
|--------|------|
| C1 語意標記 | 全部互動元件 aria-label、圖片 alt、HP/toast 加 aria-live |
| C2 鍵盤 | 數字鍵 1–4 選答案、Enter 推進劇情、Esc 關彈窗、可見 focus 樣式 |
| C3 動效尊重 | `prefers-reduced-motion` 停用漂浮／彈跳動畫 |
| C4 對比抽查 | text-dim 於玻璃面板上的對比度 ≥ 4.5:1 |

- **驗收**：純鍵盤完整打完一場戰鬥；Lighthouse Accessibility ≥ 90。

### Phase D：學習功能深化（工作量：中～大）
| 工作項 | 說明 | 優先 |
|--------|------|------|
| D1 錯題本 | 新頁面列出 box1／錯多於對的字，一鍵發起「錯題複習戰」（SRS 資料已存在，只補 UI） | 高 |
| D2 每日任務 | 地圖顯示「今日學 N 新字／複習 M 到期字」，完成給金幣（掛在現有每日獎勵系統上） | 中 |
| D3 口說跟讀 | SpeechRecognition 發音練習（僅 Chrome/Edge，標記**實驗性**，不支援則隱藏入口） | 低 |

- **驗收**：錯題本與儀表板數據一致；每日任務完成有獎勵回饋。

### Phase E：品質驗證與發布（工作量：中，穿插各階段後執行）
| 工作項 | 說明 |
|--------|------|
| E1 自動回歸 | Puppeteer：5 視口 × 全畫面截圖 + 自動打完一場完整戰鬥（含答題） |
| E2 Lighthouse | Performance / Accessibility / Best Practices / PWA 四項分數留檔 |
| E3 發布 | commit + push，GitHub Pages 線上實測（含手機安裝） |

## 五、執行順序與里程碑

```
Phase 0 → A → B → (E 驗證) → C → (E 驗證) → D → E 總驗收
```

- **里程碑 1**（0+A）：細節無瑕版 — 零 404、可靜音、emoji 補齊
- **里程碑 2**（B）：★ App 化 — 離線可玩、可安裝、零單點故障
- **里程碑 3**（C）：無障礙達標 — 鍵盤可玩、a11y ≥ 90
- **里程碑 4**（D）：學習閉環 — 錯題本 + 每日任務
